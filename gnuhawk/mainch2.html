Chapter 2
Installing GNUHAWK from RPM

This chapter describes the process of installing the GNUHAWK library and Components from pre-built RPMs.

To install GNUHAWK, users must have root or sudo access.

2.1 Dependencies

GNUHAWK has runtime and build dependencies.

2.1.1 Runtime Dependencies

The following packages must be installed to run GNUHAWK.

  • REDHAWK Core Framework 1.10.0
  • REDHAWK BULKIO Libraries 1.10.0
  • FFTW 3.2.1 or newer (This includes both the FFTW and FFTW-devel rpms.)

    For more information about installing REDHAWK, refer to the Installation chapter of the REDHAWK Manual.

2.1.2 Build Dependencies

The following dependencies are required to build the underlying VOLK GNU Radio library.

  • cmake
  • gsl-devel
  • orc-devel
  • python-cheetah
  • fftw-devel (for 64-bit architectures)

2.2 Installing GNUHAWK Library from RPM

RPM installation is for 64-bit architectures only. For 32-bit architectures, the GNUHAWK library and Components must be installed from source due to the need to create a specialized build of FFTW.

2.4 Location of Installed Files

  • GNUHAWK library is installed in $SDRROOT/dom/deps/gnuhawk/lib.
  • GNU Radio’s VOLK (Vector-Optimized Library of Kernels) library is installed in
    $SDRROOT/dom/deps/gnuhawk/lib.
  • GNUHAWK Components are installed in $SDRROOT/dom/components/gnuhawk. The following subdirectories contain Components that use GNU Radio blocks.
    • atsc
    • digital
    • fft
    • filter
    • gr
    • gr/coding
    • gr/converter
    • gr/demodulation
    • gr/ecc

    • gr/fmdet_cf
