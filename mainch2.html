Chapter 2
Installation

2.1 Introduction

Installation of REDHAWK consists of a few basic steps:

  1. Installing Dependencies
  2. Installing the Core Framework
  3. Installing the IDE
  4. Setting Environment Variables
  5. Configuring omniORB

Root permissions are necessary for configuring omniORB and may be necessary for installing the Core Framework (depending on the permissions of the installation target directory).

It is recommended that users install from Red Hat Package Managers (RPMs), but advanced users who desire custom functionality may also install from source. If installing from source, consult Section 2.5; if installing from RPMs, skip Section 2.5.

REDHAWK installation is supported for Red Hat Enterprise Linux or Community Enterprise Operating System (CentOS) versions 5 and 6, for both 32-bit and 64-bit versions. Note that CentOS version 6.4 is not supported due to a known bug in this version at the time of this writing.

2.3 Installing Dependencies

REDHAWK has several third-party dependencies. The list of dependencies may appear daunting; however, installation can be accomplished via a few yum commands. Note that multiple dependencies may be passed to yum in a single command for convenience. The lists of dependencies have been formatted so that they can be easily copied into a terminal.

2.3.1 Dependencies Packaged with Red Hat Enterprise Linux / CentOS

The following dependencies must be installed:

  • gcc \
  • gcc-c++ \
  • apr \
  • apr-devel \
  • apr-util \
  • apr-util-devel \
  • expat-devel \
  • autoconf \
  • automake \
  • libtool \
  • e2fsprogs-devel \
  • zip \
  • xsd \
  • boost \

2.4 Installing the Framework from RPMs

The REDHAWK Core Framework can be installed via either the rpm or yum commands. The specific options provided to these commands will vary depending on the configuration of the system hosting the installation. As with the included third-party dependencies, these packages may be installed conveniently by passing *.rpm to either yum or rpm when in the directory containing the packages.

The following packages are to be installed:

  • bulkioInterfaces-<version>.<distribution>.rpm
  • redhawk-sdrroot-dev-mgr-<version>.<distribution>.rpm
  • GPP-<version>.<distribution>.rpm
  • GPP-profile-<version>.<distribution>.rpm
  • redhawk-sdrroot-dom-mgr-<version>.<distribution>.rpm
  • redhawk-sdrroot-dom-profile-<version>.<distribution>.rpm
  • redhawk-<version>.<distribution>.rpm

Where <version> corresponds to the particular REDHAWK release being installed, and <distribution> corresponds to the OS version and architecture of the installation host.

The installation of the following packages is optional:

  • GPP-<version>.src.rpm
  • redhawk-<version>.src.rpm
  • bulkioInterfaces-<version>.src.rpm
  • redhawk-debuginfo-<version>.<distribution>.rpm
  • bulkioInterfaces-debuginfo-<version>.<distribution>.rpm

2.5 Installing the Framework from Source

In order to install the Core Framework from source, the redhawk-src-1.8.2.tar.gz must be downloaded.

Installing from source provides the advantage of allowing the user to set a variety of compile-time configuration options, including:

  • Disable log4cxx support
  • --disable-log4cxx
  • Enable LOG_TRACE statements
  • --enable-trace
  • Enable persistence support:
  • --enable-persistence=<persist_type>
  • (Support types: bdb, gdbm, sqlite, none)
  • Disable framework java support:
  • --disable-java
  • Change the ossie root directory
  • --with-ossie=<dir>
  • Change the SDR root directory
  • --with-sdr=<dir>

2.6 Setting Environment Variables

The environment variables SDRROOT and OSSIEHOME must be set in order to run REDHAWK. OSSIEHOME points to the location of installed libraries, header files, and executables. SDRROOT is the installation target of user-developed resources (e.g., Components and Waveforms), which implies that the developer must have write permissions to SDRROOT and its subdirectories.
