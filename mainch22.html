Chapter 22
Exploring a Running Domain Using the REDHAWK IDE

The Domain Manager, as described in Section 11.2, contains knowledge of all existing CORBA-enabled objects installed or loaded onto the system. This includes references to all file systems (through the File Manager), Device Managers, and all Applications (and their Resources). The Domain Manager is seen as the central bookkeeper. The REDHAWK IDE can be used to run or connect to a running Domain, view the contents of a running Domain, and launch and interact with Applications within the Domain.

22.1 Connecting to a Domain

The IDE provides the options of either launching a Domain through the IDE interface or connecting to a Domain that has been launched outside of the IDE.

22.1.1 Launching and Connecting Using the IDE

To launch and connect to a Domain using the IDE:

  1. Right-click the Target SDR from the SCA Explorer view and select Launch…


    Figure 22.1: Right-clicking the target SDR will bring up a menu allowing the Domain to be launched

  2. Choose a name for the Domain Manager
  3. Select a Device Manager to start. While this step is optional, a running Device Manager is required to launch Applications.

    Optionally, the debug level may be set. By default it is set to info which will display any messages at the info level or higher which includes: info, warning, error, and fatal message levels. If this is the first time using REDHAWK, changing the level from Info to Debug for both the Domain and Device Manager may be helpful in the learning process.

  4. Finally, click OK to launch and connect to the new Domain.
  5. Notice that the IDE reacts to the newly launched Domain:
    1. The new Domain has been added to the SCA Explorer view
    2. The new Domain, within a short amount of time, should have connected and will indicate this to the right of the Domain name within the SCA Explorer view.
    3. At least one new Console view (within the IDE) have been created. One contains a nodeBooter instance for the Domain Manager that launched and one nodeBooter instance for each Device Manager.


Figure 22.2: Console showing the Domain Manager start up with debug logging and the available Device Manager console

22.1.2 Connecting to a Running Domain

How a Domain and a Device Manager can be launched from the command line is shown in Section 11.2.1.

To connect to this Domain through the IDE:

  1. Click the New Domain Connection button (i.e. the plus symbol) in the upper right of the SCA Explorer View.
  2. Provide the Domain name and the running Naming Service’s location. If unsure of the Naming Service location, use the default.
  3. Click one of the options to:
    • Don’t connect: This will add the Domain to the SCA Explorer View but will leave the Domain in the disconnected state. When the IDE is restarted, the Domain will remain in the SCA Explorer and will be in the disconnected state. After adding a disconnected Domain to the SCA Explorer view the Domain may be connected by right clicking on the Domain and selecting Connect.
    • Connect Once: This will add the Domain to the SCA Explorer View and will connect the IDE with the Domain. When the REDHAWK IDE is restarted, the Domain will remain in the SCA Explorer but will be in the disconnected state.
    • Always Connect: This will add the Domain to the SCA Explorer View and will connect with IDE with the Domain. When the REDHAWK IDE is restarted, the Domain will remain in the SCA Explorer and will attempt to connect with this Domain.


    Figure 22.3: New Domain Manager dialog set to connect once

  4. Select Finish to close the wizard.
  5. The Domain should now appear in the SCA Explorer View. If Connect Once or Always Connect was chosen, the Domain should have connected. If Don’t Connect was selected, right click on the Domain and select Connect.

    Many of these options may be changed later through the Properties View.

22.2 Viewing the Contents of the Domain in the SCA Explorer View

Once the Domain has been connected, a user may look at the file system visible to the Domain Manager and its attached Device Managers. If the Properties View is open, note that clicking through the items within the Domain Manager shows detailed information about each item.

The Domain Manager’s root contains three folders:

  • Device Managers: Shows the currently connected Device Managers. More than one Device Manager may be connected to the Domain. Each Device Manager entry consists of a single Node and where each Node may contain multiple Devices. Right-click on Devices to monitor Port information, plot Port output and play audio.
  • SCA Waveforms: This folder contains the running Applications on the Domain. As Applications are launched, they appear here and can be expanded to show each of the running Components within the Application. These Components can be expanded to show the Device they are executing on and Port information.
  • File Manager: The File Manager shows all the file systems in the Domain’s view. It contains references to all Components, Devices, Waveforms, and the Device and Domain Managers configuration and executable files.

22.3 Launching an Application

In this section, it is assumed that a running Domain Manager and Device Manager exist and that the Domain Manager contains a working Waveform.

To launch an Application:

  1. Right-click on the Domain and select Launch Waveform….
  2. In the Launch SCA Waveform Wizard dialog, select the Waveform to launch.
  3. If there are no additional settings to specify, click Finish to immediately launch the Application. For completeness, we will step through the additional pages of the dialog.
  4. On the second page, the user has the chance to set the Properties of the Components within the Application. Any Property modified here is specific to this Application and will not impact the Component‘s execution in other environments. Notice that as the Properties are changed from their default values the now non-default values appear in bold as shown in Figure 22.3.
  5. The third, and final page, allows the user to specify what executable Device each of the Components will launch on. If left on Auto, REDHAWK will determine the executable device based on any allocation Properties and dependencies set on the Components and Devices.
  6. Click Finish to launch the Application.

    Notice that the Domain Manager within the SCA Explorer view now shows the launched Application within the SCA Waveforms folder.

  7. Unless the user has specified to start the Application on launch in the Launch SCA Waveform Wizard, the Application has not been started.

    There are multiple options for starting the Application.

    • Click the Start SCA Waveform (i.e. the green arrow) button located above the Waveform Editor.
    • Right clicking the Waveform from the Domain’s SCA Waveform folder within the SCA Explorer view and selecting Start.
    • Or right-clicking on the Component from within the Waveform Editor and selecting Start.
  8. Regardless of how the Application is started, the Components within the Application should now have a green glow in the upper portion indicating that they are in the started state.
  9. The SCA Explorer view should also indicate that the Application has started by displaying “STARTED” next to this Application’s instance.
  10. When finished, the Application may be stopped and released using any of the methods listed to start the Application except this time selecting either Stop then Release.

    Note that it is not necessary to select stop prior to releasing the Application. The IDE will try to stop the Application before releasing.


Figure 22.5: Application Properties turn bold when non-default values are set

22.4 Plotting BulkIO Ports

The REDHAWK IDE contains the ability to plot using the NeXtMidas plotting framework. If the output Port uses the BulkIO interface, it can take advantage of this feature and plot a line graph or a falling raster.

To bring up a plot within the IDE:

  1. Make sure that the Component is currently in the started state.
  2. Right-click on the desired Port to plot.
  3. Select either Plot Port Data or Plot Port FFT.

    A new view is created, and it contains the plot of the Ports output data.

The new view has the same name as the source Port. To view additional source information, hover the mouse over the title.

For more information on how to use the Plot View, refer to Section 19.4.6.

22.5 Displaying Link Statistics

In addition to Port plotting, a user may want to monitor the amount of data flowing out or into a particular Port. These link statistics are helpful when debugging and can help identify which Component is slowing down or dropping information during data processing.

To bring up the Port Monitor view:

  1. Make sure that the Component to be monitored is currently in the Started state.
  2. Right click on the Port to monitor.
  3. Select Monitor Port.

    This will bring up a new view and display the following information:

    • Name: The name of the Port or Port connection.
    • Elements/sec: The rate of CORBA elements transferred in the pushPacket data call.
    • MBps: Mega Bytes transferred per second.
    • calls/sec: Number of push calls per second to the Port.
    • Stream IDs: List of all active stream IDs.
    • Avg. Queue Depth: For Components that queue data before processing/sending, the average queue depth measured as a percentage. If a Port does not queue data this value is set to zero.
    • Time: The elapsed time, in seconds, since the last packet was transferred via a push packet call.

For more details about the Port Monitor view, refer to Section 19.4.9.

22.6 Getting Details About Error Conditions

If an error condition occurs within the Domain Manager which prevents a Component, Device, Domain Manager, or Device Manager from running correctly, the objects representation in the SCA Explorer will be marked with a decorator in the lower left corner. Mouse hovering over the items icon will provide a short description of the issue; however, if more than one problem has occurred, the hover text will read “Multiple Problems exist with this item”.


Figure 22.6: Mouse hovering over items with error decorators will often provide information about the error

More detail about an error can be found within the Properties view of the item.

To view the details about an error condition:

  1. With the item selected, select or open the Properties View.
  2. From the Properties View, select the Advanced tab
  3. Select the status row. This should cause the Details button to appear.
  4. Click Details to bring up a detailed dialog of the current error conditions.
