Chapter 6
Launching the RBDS Waveform on the BeagleBone Black

This chapter explains how to run the RBDS Waveform on the BeagleBone Black.

6.1 Connecting the IDE to the BeagleBone Black REDHAWK Domain

The following procedure explains how to connect the IDE to the BeagleBone Black REDHAWK Domain.

  1. Start the Domain Manager on the BeagleBone Black.
  2. Start the IDE on the development computer.
  3. In the SCA Explorer view, click New Domain Connect.
  4. In the Domain Name section, enter REDHAWK_DEV.
  5. In the Name Service Initial Reference section, enter corbaname::<beaglebone black IP>.
  6. Click Finish.

6.2 Launch the Device Manager and the RBDS Waveform

  1. On the BeagleBone Black, to launch the rtlNode, enter the following command: The REDHAWK Device Manager is launched.
  2. On the development computer, in SCA Explorer, expand the Domain Manager, expand Device Managers, and verify the rtlNode is displayed.

    If the Domain is not displayed or not connected, refer to Section 6.1.

  3. Right-click the Domain Manager and select Launch Waveform.
  4. Select the RBDS_wf Waveform, click Next, and click Next again.
  5. Click Finish and run the Waveform.

    The RBDS Waveform and Components are now properly launched and running on the BeagleBone Black. The RBDS Messages are connected to the Event Channel, RBDS_Channel.

If messages are not displayed in the eventviewer terminal, you may need to manually set the gain for the RTL Device. (The messages may not be displayed because the signal strength is too high, which can overwhelm the RTL Automatic Gain Control.) First, you must verify data is flowing into the RBDSDecoder. In the running Waveform, right-click the dataShort_out Port from the DifferentialDecoder in the Waveform, and select Plot Port Data. If data is flowing into the RBDSDecoder, then use the following procedure to set the gain manually for the RTL Device.

  1. In the IDE, in the SCA Explorer view, select DeviceManagers, rtlNode, RTL2832U, FrontEnd Tuners, RX_DIGITIZER.
  2. Left-click the allocated tuner, and from the Properties tab, set the Value for the AGC Property to False.
  3. Then set the Value for the Gain Property to 0.0.

If running a Node on the BeagleBone Black and a Node on the development computer, the Device’s Nodes may have the same name (for example, GPP). This causes problems when you select the Devices on which to run certain Components.
