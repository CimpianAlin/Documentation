Chapter 3
Building and Installing GNUHAWK from Source

The GNUHAWK source baseline consists of a compatibility library of GNU Radio base classes, along with GNU Radio version 3.6.2. The source tree utilizes GNU’s Auto Tools to configure and build the compilation environment for the compatibility library, the VOLK library, the GNU Radio Blocks, and their supporting files.

To install GNUHAWK, users must have root or sudo access.

3.1 Dependencies

GNUHAWK has runtime and build dependencies.

3.1.1 Runtime Dependencies

The following packages must be installed to run GNUHAWK.

  • REDHAWK Core Framework 1.10.0
  • REDHAWK BULKIO Libraries 1.10.0
  • FFTW 3.2.1 or newer (This includes both the FFTW and FFTW-devel rpms.) For more information about installing REDHAWK, refer to the Installation chapter of the REDHAWK Manual.

3.1.2 Build Dependencies

The following dependencies are required to build the underlying VOLK GNU Radio library.

  • cmake
  • gsl-devel
  • orc-devel
  • python-cheetah
  • fftw-devel (for 64-bit architectures)
  • FFTW (for 32-bit architectures)
    FFTW must be configured with the \variable{-{}-enable-sse} flag. This requires installing FFTW from source with the following configure call: \vspace{12 pt} \begin{terminalCode} ./configure --enable-float --enable-sse --enable-shared --disable-static \end{terminalCode} For FFTW to install properly, certain environment variables need to be set in \filename{.bashrc}. \begin{itemize} \item For installation on 32-bit architecture, set the following environment variables: \begin{terminalCode} export LD_LIBRARY_PATH=/usr/local/lib:$LD_LIBRARY_PATH export PKG_CONFIG_PATH=/usr/local/lib/pkgconfig:$PKG_CONFIG_PATH \end{terminalCode} \item For installation on Ubuntu, set the following environment variables: \begin{terminalCode} export LD_LIBRARY_PATH=${OSSIEHOME}/lib64:$LD_LIBRARY_PATH export PKG_CONFIG_PATH=/usr/local/lib/pkgconfig:$PKG_CONFIG_PATH \end{terminalCode} \end{itemize}

3.2 Directory Structure of GNUHAWK Source

The GNUHAWK source is structured as follows:

  • components contains the GNUHAWK Components.
  • include contains the GNUHAWK include files installed to support the GNUHAWK library.
  • gnuradio contains the GNU Radio baseline, version 3.6.2.
  • qa/components contains the supporting Components for QA tests.
  • qa/gnuradio contains the GNUHAWK version of the GNU Radio python module needed to perform QA testing.
  • qa/tests contains the QA tests to be run, most of which are unmodified from the original GNU Radio tests.
  • src contains the source code needed to build the library, including Makefile.am files used to build the original GNU Radio blocks and supporting classes.

3.5 Location of Installed Files

  • GNUHAWK library is installed in $SDRROOT/dom/deps/gnuhawk/lib.
  • GNU Radio’s VOLK (Vector-Optimized Library of Kernels) library is installed in
    $SDRROOT/dom/deps/gnuhawk/lib.
