IDE Quickstart

This section provides a simple example of the REDHAWK signal processing development environment, though it is far from a comprehensive guide to all of the features available in the REDHAWK IDE. While this guide assumes the user has no prior knowledge of the REDHAWK Core Framework, certain concepts and skills are required to fully understand the material.

The minimum technical requirements include:

  • Object-oriented programming experience
  • Linux/Unix experience
  • Engineering/Computer Science background
  • Understanding of basic communication theory

In addition to the minimum set of technical requirements, the following prerequisites must be met before beginning the following procedure:

  • REDHAWK Core Framework and IDE installed
  • Example Components installed:
    • Signal Generator
    • Hard Limit

Basic IDE Use

The following procedure discusses how to launch the REDHAWK IDE, how to create a signal generator, and how to test the input/output response of a Component.

To launch the REDHAWK IDE:

  1. Assuming the REDHAWK IDE is installed in ~/redhawk-ide/, start the IDE with the following commands:
  2. If prompted to specify a workspace location, select an appropriate location and select OK.

    If this is the first time running the IDE using this workspace, the Welcome screen is presented and prompts you to configure PyDev. If this is the case:

    1. Click Auto to allow PyDev to automatically detect your configuration.
    2. Click Workbench to close the splash screen and continue into the IDE Workbench.
  3. From the SCA Explorer view expand Sandbox, and double-click Chalkboard.

To create a signal generator:

  1. From the Chalkboard Palette, drag the SigGen Component into the Chalkboard canvas.
    1. If prompted to Select Implementation, select Python, and click OK.
  2. Right-click the SigGen Component, and click Start.
  3. Right-click the SigGen Component, and click Show Properties View.
  4. From the Properties view, change the frequency to 20Hz.
  5. Right-click SigGen’s out Port, and click Plot Port Data.
  6. Right-click the SigGen Component, and click Stop.

To test the input/output response of a Component:

  1. From the Chalkboard Palette, drag the HardLimit Component into the Chalkboard canvas.
    1. If prompted to Select Implementation, select Python and click OK.
  2. Click-and-drag from SigGen’s out Port to the HardLimit input Port.
  3. Right-click the SigGen Component, and click Start.
  4. Right-click the HardLimit Component, and click Start.
  5. Right-click the HardLimit’s output Port, and click Plot Port Data.

    Note: The \guilabel{Plot Port} view now has two tabs at the bottom left, one for each of the plotted Ports.
  6. Right-click the SigGen Component, and click Show Properties View.
  7. From the Properties view, change the magnitude to 5.

    The Plot Port view for the HardLimit output Port is now limiting the output to 1.
