Chapter 23
SCA Explorer

The REDHAWK SCA Explorer, an application built on the REDHAWK Core Framework, is used to navigate the contents of an SCA Domain. It provides capabilities for viewing the contents of the Domain, configuring Domain resources, and launching Waveforms in the Domain.

The SCA Explorer product is a subset of the Eclipse IDE; it is meant to provide a minimal set of plug-ins necessary to create an Eclipse application that interacts with REDHAWK.

The SCA Explorer is meant as a starting point only. Most applications should productize their own set of plug-ins based on those of the SCA Explorer, in order to fit their own needs.

23.1 Using SCA Explorer

The SCA Explorer is a full-fledged application and does provide some basic functionality.

In order to run the SCA Explorer:

  1. Obtain a copy of SCA Explorer and unzip to any location.
  2. Run the executable


Figure 23.1: SCA Explorer

For those familiar with the REDHAWK IDE many of the same tools will be available in the SCA Explorer. The SCA Explorer View and the CORBA Name Browser are shown, as well as the Properties View and the Error Log View. All of these views function the same way as in the IDE.

Unlike a typical Eclipse application, these views can neither be moved nor closed. This functionality is intentional as the SCA Explorer is designed to present as few features as possible in an effort to not confuse the user.

This behavior is accomplished with the use of a fixed perspective.

23.2 Creating a Custom REDHAWK GUI for the SCA Explorer

The SCA Explorer is a great building block, but it does not always provide the look and feel preferred by the end user. Therefore it is possible to create custom applications based off of the SCA Explorer product.

To create a custom application:

  1. Click Window > Preferences
  2. In the Preferences dialog, expand the Plug-in Development select Target Platform

    Figure 23.2: Target Platform Preference Page

  3. Click Add…
  4. In the New Target Definition window, select Nothing: Start with an empty target definition, click Next
  5. Enter a name for the target or leave as is
  6. Click Add…
  7. Select Installation, click Next
  8. Enter the location of the SCA Explorer product, click Finish
  9. Click Finish
  10. Select the checkbox next to the new target platform you just added, click OK

For help related to developing Eclipse applications or productizing an application refer to Appendix A.
