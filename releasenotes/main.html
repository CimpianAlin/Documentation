REDHAWK Version 1.10.1 Release Notes

The effort in REDHAWK Version 1.10.1 focused on fixing bugs and improving the reliability of the core framework and IDE.

Core Framework Fixes and Improvements

  • Fixed a resource starvation issue with the Device Manager and Domain Manager.
  • Fixed issue that prevented updating Components from 1.8 to 1.10.
  • Fixed issue that caused a deadlock in certain Application creation failures.
  • Fixed issue that caused remote Domain registrations to not be written to the persistence store.
  • Fixed issue that caused the AllocationManager to return incorrect values.
  • Fixed issue that caused improper handling of individual Property failures in Python query().
  • Improved the performance in Python Property handling.
  • Added the ability for the same Property change listener method in Python to monitor multiple Properties.
  • Fixed issue that caused the LD_LIBRARY_PATH and OCTAVE_PATH to update incorrectly in C++ Loadable Devices.
  • Fixed issue that caused a resource leak in C++ Loadable Devices.
  • Fixed issue that caused some Component processes to be orphaned and exhaust resources.
  • Fixed issue that caused the system to exit if an ApplicationFactory was not found in createApplication().
  • Restored public access to currentSRIs for C++ output Ports.
  • Fixed issue that caused a potential deadlock situation between getCurrentQueueDepth() and pushPacket() for Python input Ports.
  • Fixed issue that caused a potential deadlock situation on last packet for C++ input Ports.
  • Fixed issue that caused thread safety to not be enforced in connections() and statistics() for Java output ports.
  • Fixed issue that caused active stream IDs from Python input Ports to not be reported in statistics.
  • Improved thread safety for Octave Components.
  • Fixed bugs in creating Octave Components with Simple Sequence Properties.
  • Fixed parsing of logging config file for setting Component working directory with Octave Components.
  • Fixed issue with the build.sh script that caused C++ Components or Devices to not install properly.
  • Fixed the test to determine if the OS version dependency matches the Device OS version.
  • Fixed the timestamp used for the logging of events for C++ Components. The timestamp was being set to the time that the application was started instead of the actual time of the event.
  • Fix provided to log4py to properly support log4cxx configuration options, which can begin with upper or lower case letters. Prior to the fix, only options that started with lowercase letters were recognized.
  • Fixed host collocation problem involving Components that contained soft package dependencies.
  • Fixed memory leaks caused by using TypeCode_ptr instead of TypeCode_var for variables.

IDE Fixes and Improvements

  • Fixed permission error with IDE plotting in the Python Sandox. The error occurred only when the IDE was installed via RPM.
  • Fixed issue that caused the Chalkboard to be opened multiple times.
  • Fixed issue that caused errors to occur when releasing Components/Waveforms in the Sandbox.
  • Fixed issue that caused errors in the IDE when connecting external Ports within the IDE Sandbox.
  • Fixed issue that caused Waveforms to timeout and not launch when launching multiple Waveforms in the Sandbox at the same time.
  • Fixed issue that caused the build.sh script created at the root of a Component or Device project to be overwritten by the IDE.
  • Fixed issue that caused the DataList for a BULKIO Port to not correctly handle complex data.
  • Fixed issue that caused the FrontEnd Interface Wizard to not consistently present the default options.
  • Fixed issue that caused only one tuner to be deallocated when Deallocate All was selected from the context menu of a FrontEnd Device.
  • Fixed issue that caused the IDE to not display an error message if a FrontEnd tuner allocation failed.
  • Fixed issue that caused an error in the IDE when comments were added to the XML editor for a Waveform (SAD file).
  • Fixed issue that allowed users to create an invalid Waveform name when creating a new Waveform project.
  • Fixed issue that caused an error when users attempted to launch a Domain before the SDR and IDL libraries were loaded.
  • Fixed issue that caused the IDE to generate incorrect XML for a struct sequence if the kind (configurationkind) was not selected.
  • Fixed issue that caused the IDE to fail to reflect Domain activity if many Waveforms were torn down in rapid succession.
  • Fixed issue that caused the IDE to fail to include execparams when launching Components.
