Chapter 5
Using GNUHAWK

After you install REDHAWK and GNUHAWK, GNUHAWK Components provide the same functionality within the IDE and Sandbox that standard REDHAWK Components provide. For more information about using Components, creating Waveforms, using the IDE, and using the Sandbox, refer to the REDHAWK Manual.

GNUHAWK 1.10 introduces a new feature for GNUHAWK Components, the ability to set Stream IDs.

The ability to set Stream IDs does not currently exist for standard REDHAWK Components.

5.1 Setting Stream IDs

The default stream ID for all Components is <SID>_<port number>.

If Components have multiple output Ports, it may be difficult to determine what Components a stream has traveled through at any given point. Therefore, for GNUHAWK Components with one input Port and multiple output Ports, you may want to set customized stream IDs. You set the customized stream ID by using the struct sequence Property, stream_id_map. This struct sequence consists of the struct, stream_id_struct, which has the following two simple string Properties.

  • port_name, the name of the Port with the custom stream ID.
  • stream_id, the stream ID value to be set. A custom string may be used, or you can use the following helpful shortcuts:
    • %D produces the port index number (example: 0_)
    • %SID produces the stream ID from the incoming port (example: sig_source_0_)
    • %C produces the Component’s instance name (example: stream_to_streams_ff_2o_)

    If this Property is not set for any of the out Ports, the default stream ID is used.
