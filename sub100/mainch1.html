Chapter 1
Introduction

Sub$100 is an example REDHAWK implementation project that demonstrates the powerful capabilities of REDHAWK on a constrained budget. The goal of this project is to show that REDHAWK can execute beneficial, real-time processing applications on a platform with limited resources. This project uses REDHAWK on a BeagleBone Black to receive an FM radio signal using an RTL Device; however, you could also use this hardware to process other types of RF signals, such as ADS-B data, GPS data, or images from the NOAA weather satellites. The REDHAWK application then recovers the RBDS data (Artist, song, station, etc.) and outputs this data to the user. The total equipment cost is less than $100.

1.1 What is REDHAWK?

REDHAWK is a software defined radio (SDR) framework designed to support the development, deployment, and management of real-time software radio applications. To support the design and development of software applications, REDHAWK provides tools that allow development and testing of software modules (DSP processing blocks) called "Components" and composition of Components into "Waveform Applications" that can be seamlessly deployed on a single computer or multiple network-enabled computers. The Component-based nature of REDHAWK allows easy code reuse within a Waveform as well as between different Waveforms. Using the REDHAWK IDE, developers can create new Components and arrange them into Waveforms, making connections between Components in a very visual way by using the input and output ports of the Components. REDHAWK uses a middle ware technology to provide seamless data exchange between different processor architectures and different software languages. For more information about REDHAWK, refer to the REDHAWK User Manual.

1.2 What is an RTL Device?

RTL Devices are Digital Video Broadcast-Terrestrial (DVB-T) USB devices originally intended to allow computer reception of television in Europe. However, the Devices have been reverse-engineered by the SDR community to act as a cheap stand-in for more expensive Devices such as the USRP and Matchstiq, albeit only with reception capabilities. Because the tuner allows configuration of parameters, such as the center frequency to tune to, the sample rate at which to sample the analog signal, and the gain to amplify or attenuate the signal by, these devices serve as an adequate FrontEnd Device for an SDR project. The frequency range of the Devices range from 20 MHz up to around 2 GHz, allowing users to receive data such as FM Radio, GPS, Air Traffic data, and many others.

1.3 What is the BeagleBone Black?

The BeagleBone Black is an embedded development board similar to the Raspberry Pi. With an ARM processor, internet connectivity, and USB hosting capabilities, it can serve as a PC if need be, although a simple one. Because the BeagleBone Black is capable of running Linux, it is also capable of running the REDHAWK framework. Because of REDHAWK’s many dependencies, it is desirable to use a distribution of Linux with an existing build for the BeagleBone Black as well as with a package management system capable of providing these dependencies. Ubuntu was chosen because a pre-built BeagleBone Black image now exists and due to the robustness of its package manager, aptitude.

1.4 What is the Radio Broadcast Data System (RBDS)?

The Radio Broadcast Data System (RBDS) is the U.S. version of the Radio Data System (RDS) European standard. These standards are intended to provide supplemental information to FM broadcast radio listeners such as meta information about the audio or traffic data. The spectrum of a typical FM broadcast after tuning to baseband is shown below:


The Mono data is seen from about 0 Hz up to 15 kHz, and the pilot tone sits at 19kHz. The left and right audio channels are between 23 kHz and 53 kHz. Finally, the RBDS information is centered at 57 kHz.

1.5 How REDHAWK Works with a BeagleBone Black and an RTL Device to Make an RBDS Receiver

In this SUB$100 project, the BeagleBone Black coordinates the interaction between the FrontEnd Device and the Components executing on the processor, as well as the execution of the REDHAWK Domain Manager. The RTL collects the data, and REDHAWK runs on the BeagleBone Black and processes the data. This manual explains the steps required to complete the Sub$100 project.

After completing the tasks in this manual, a REDHAWK Domain Manager, Device Manager and an RBDS Waveform will be running on the BeagleBone Black. The REDHAWK framework distributes the RBDS Components executables (built for the armv7l processor) to one or more BeagleBone Black Nodes. The RBDS Waveform uses an RTL-SDR dongle to tune to an FM radio station, receives and decodes the RBDS data (Station, Artist, Song Name, etc.), and sends that information as a Message over an Event Channel. For more information on Events and Messages, refer to chapters 16 and 17 in the REDHAWK User Manual. The development computer will have the ability to passively listen for received messages and will be able to examine and plot output data of the RBDS Waveform Components using the stand-alone REDHAWK IDE.

1.6 Prerequisites for the Sub$100 Project

The SUB$100 project requires the following hardware.

  • 1 Linux-based development computer
  • 1 BeagleBone Black
  • 1 MicroSD card (> 4GB, <= 32 GB)
  • 1 MicroSD card reader
  • 1 powered USB hub
  • 1 RTL USB dongle
  • 1 210-460mA@5V DC source OR 1 MiniUSB cable (Power)

The RTL USB dongle must be connected to the BeagleBone Black via the powered USB hub because the USB port on the BeagleBone Black does not provide sufficient power to operate the dongle.

Before you begin, the development computer must have the REDHAWK IDE and JAVA 1.7.0 installed. For more information about installing the REDHAWK IDE, refer to the REDHAWK Manual, Version 1.10.0, Section 2.5 Installing a Stand-alone IDE.

1.7 Creating the Sub$100 Project

To create the Sub$100 project, you must complete the following tasks:

  1. Set up the microSD card for the BeagleBone Black
  2. Install REDHAWK on the BeagleBone Black
  3. Install the RBDS Waveform Dependencies on the BeagleBone Black
  4. Prepare the Waveform Runtime Environment
  5. Launch the RBDS Waveform on the BeagleBone Black
