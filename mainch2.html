Chapter 2
Installation

2.1 Introduction

This chapter discusses the process of installing the Core Framework, the IDE, and the Basic Components. The Core Framework is the software back-end of REDHAWK. The IDE is a Graphical User Interface (GUI) for development and use of REDHAWK systems. The Basic Components are building blocks that developers can use to create radio algorithms.

Complete installation of REDHAWK consists of a few basic steps:

  1. Installing Dependencies
  2. Installing the Core Framework
  3. Installing the IDE
  4. Installing Basic Components
  5. Setting Environment Variables
  6. Configuring omniORB

Root permissions are necessary for configuring omniORB and may be necessary for installing the Core Framework (depending on the permissions of the installation target directory).

It is recommended that users install from Red Hat Package Managers (RPMs), but advanced users who desire custom functionality may also install from source. If installing from source, consult Section D after installing dependencies.

REDHAWK installation is supported for Red Hat Enterprise Linux or Community Enterprise Operating System (CentOS) versions 5 and 6, for both 32-bit and 64-bit versions. A developer install of Red Hat Enterprise Linux or CentOS is recommended.

2.2.1 Developer Dependencies

Building REDHAWK assets requires several other development dependencies which must be installed in order to compile new components.

The following packages must be installed in order to develop and build new REDHAWK assets.

  • apr-devel \
  • apr-util-devel \
  • autoconf \
  • automake \
  • expat-devel \
  • gcc-c++ \
  • "java-devel >= 1.6 "\
  • libtool \
  • numpy \
  • python-devel

The following dependencies are required for Red Hat/CentOS 5:

  • e2fsprogs-devel \
  • python-uuid

The following dependencies are required for Red Hat/CentOS 6:

  • libuuid-devel \
  • boost-devel
