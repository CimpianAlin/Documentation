Chapter 2
Installation

2.1 Introduction

This chapter discusses the process of installing the Core Framework, the IDE, and the Basic Components. The Core Framework is the software back-end of REDHAWK. The IDE is a Graphical User Interface (GUI) for development and use of REDHAWK systems. The Basic Components are building blocks that developers can use to create radio algorithms.

Complete installation of REDHAWK consists of a few basic steps:

  1. Installing Dependencies
  2. Installing the Core Framework
  3. Installing the IDE
  4. Installing Basic Components
  5. Setting Environment Variables
  6. Configuring omniORB

Root permissions are necessary for configuring omniORB and may be necessary for installing the Core Framework (depending on the permissions of the installation target directory).

It is recommended that users install from Red Hat Package Managers (RPMs), but advanced users who desire custom functionality may also install from source. If installing from source, consult Section D.2.

REDHAWK installation is supported for Red Hat Enterprise Linux or Community Enterprise Operating System (CentOS) versions 5 and 6, for both 32-bit and 64-bit versions.

Note, that due to a bug in boost-1.41, REDHAWK, and REDHAWK C++ projects, will not compile when using gcc version 4.4.7 (standard in Red Hat and CentOS 6.4). In order to fix this, the boost file exception_ptr.hpp must be patched. For standard boost installations, this file is located in /usr/include/boost/ and requires root permissions to change.

2.3 Installing Dependencies

REDHAWK has several third-party dependencies; installation can be accomplished via a few yum commands. Note that multiple dependencies may be passed to yum in a single command for convenience.

REDHAWK requires dependencies that are included in the base distribution of Red Hat/CentOS or are packaged in a dependencies tarball along with the REDHAWK distribution. The EPEL repository should be added to the list of yum repositories that are searched. For instructions on how to add EPEL to the yum search, see Section C.1. The following list contains the dependencies that must be installed. The lists of dependencies have been formatted so that they can be easily copied into a terminal as arguments to a yum install command.

The following dependencies must be installed:

  • gcc-c++ \
  • apr \
  • apr-devel \
  • apr-util \
  • apr-util-devel \
  • expat-devel \
  • autoconf \
  • automake \
  • libtool \
  • python-devel \
  • numpy

The following dependency is required for Red Hat/CentOS 5:

  • e2fsprogs-devel

The following dependencies are required for Red Hat/CentOS 6:

  • libuuid-devel \
  • boost-1.41.0-1 \
  • boost-devel-1.41.0-1

The following dependency is not required, but is recommended:

  • valgrind

REDHAWK requires Python 2.4 (default on Red Hat/CentOS 5) or Python 2.6 (default on Red Hat/CentOS 6). These should already be installed with the base Linux installation, but it is worth noting that REDHAWK is not compatible with Python 3.
