Chapter 21
Exploring SDRROOT Using the REDHAWK IDE

The REDHAWK IDE provides a visualization of the file system contained within the SDR Root in the SCA Explorer view. The SDRROOT is referred to within the IDE as the Target SDR. The Target SDR location may be changed from within the IDE preferences page.

To change the target SDR:

  1. Select Window > Preferences
  2. Select the REDHAWK > SDR preference page
  3. Change the Local SDR Location value
  4. Click OK

21.1 Browsing Installed SDR Objects

From the SCA Explorer and the Target SDR, one can browse the installed Components, Devices, Nodes, Services and Waveforms. Each of these SCA Objects are placed in their own folder which may be expanded to show their content. Depending on the object type, different right-click operations and tree structure manipulation are permitted.

21.1.1 Components

Mouse-hovering over an item within the SCA Explorer will show its corresponding file and the full path to its location. Mouse-hovering over a Component shows that its representation is derived from the SPD file within the SDR Root.

One may perform the following actions from the SCA Explorer view on Component objects:

  • Local Component Launch: Right-click the Component SPD file and select Launch to start an instance of the Component within the Sandbox’s Chalkboard.
  • View within SCA Editor: To open the SPD file within an SCA Editor one may either, double-click the SPD file, right-click and select SCA Component Editor or right-click and select Open With> SCA Component Editor. This will open the SCA Component Editor, as described in Section 19.4.1, in read-only mode. One may not edit the fields or perform any code generation from this version of the editor.
  • Delete: Delete a Component from the SDR Root by right-clicking and selecting Delete. One may also use the keyboard’s Delete key to initiate a deletion. This will remove all files pertaining to this Component from the SDR Root including the SPD, PRF, SCD, and executable. The delete action also supports multiple selection. Using the Shift key, one may highlight more than one Component and use either the right-click menu or the Delete key to delete all selected Components.

    Deleting an item from the SDR Root cannot be undone

21.1.2 Devices

Mouse-hovering over an item within the SCA Explorer will show its corresponding file and the full path to its location. Mouse hovering over a Device will show that its representation is derived from the SPD file within the SDR Root.

One may perform the following actions from the SCA Explorer view on Device objects:

  • Local Device Launch: Right-click the Device SPD file and select Launch to start an instance of the Device within the Sandbox’s Device Manager. Expand Sandbox> Device Manager to view the running Device.
  • View within SCA Editor: To open the SPD file within an SCA Editor one may either, double-click the SPD file, right-click and select SCA Component Editor or right-click and select Open With> SCA Component Editor. This will open the SCA Editor, as described in Section 19.4.1, in read-only mode. One may not edit the fields or perform any code generation from this version of the editor.
  • Delete: Delete a Device from the SDR Root through the right-click Delete menu. One may also use the keyboard’s Delete key to initiate a deletion. This will remove all files pertaining to this Device from the SDR Root including the SPD, PRF, SCD, and executable. The delete action also supports multiple selection. Using the Shift key, one may highlight more than one Device and use either the right-click menu or the Delete key to delete all selected Devices.

    Deleting an item from the SDR Root cannot be undone

21.1.3 Nodes

Mouse-hovering over an item within the SCA Explorer will show its corresponding file and the full path to its location. Mouse-hovering over a Node will show that its representation is derived from the DCD file within the SDR Root.

One may perform the following actions from the SCA Explorer view on Node objects:

  • Launch Device Manager: One may launch a Device Manager onto a running Domain for this Node. Right-click on the Node and select Launch Device Manager to bring up a Launch Device Manager Dialog. From this dialog one may select from a list of running Domains. Pressing OK in this dialog will create a new Console view and run an instance of nodeBooter using this Node’s DCD file.
  • View within SCA Editor: To open the DCD file within an SCA Editor, one may either, double-click the DCD file, right-click and select SCA Node Editor or right-click and select Open With> SCA Node Editor. This will open the SCA Node Editor, as described in Section 19.4.1, in read-only mode. One may not edit the fields or add any additional Devices.
  • Expand Node: Using the tree structure to expand the Node, one should notice that the IDE can decompose the sections of the DCD file into the referenced Devices, Device Manager, Domain Manager, and Partitioning sections. By selecting these objects with the Properties View open, one can view all the information from the XML in a tabular format.
  • Delete: Delete a Node from the SDR Root through the right-click Delete menu. One may also use the keyboard’s Delete key to initiate a deletion. This will remove only the DCD file and the folder containing it from the SDR and will not delete the referenced Devices. The delete action also supports multiple selection. Using the Shift key, one may highlight more than one Node and use either the right-click menu or the Delete key to delete all selected Nodes.

    Deleting an item from the SDR Root cannot be undone

21.1.4 Services

Mouse-hovering over an item within the SCA Explorer will show its corresponding file and the full path to its location. Mouse-hovering over a Service will show that its representation is derived from the SPD file within the SDR Root.

One may perform the following actions from the SCA Explorer view on Service objects:

  • Launch: Launching the Service will run the executable on the local machine within the IDEs console view.
  • View within SCA Editor: To open the SPD file within an SCA Editor, one may either, double-click the SPD file, right-click and select SCA Component Editor or right-click and select Open With> SCA Component Editor. This will open the SCA Editor, as described in Section 19.4.1, in read-only mode. One may not edit the fields or perform any code generation from this version of the editor.
  • Delete: Delete a Service from the SDR Root by right-click Delete. One may also use the keyboard’s Delete key to initiate a deletion. This will remove all files pertaining to this Service from the SDR Root including any SPD, PRF, SCD, and executable files. The delete action also supports multiple selection. Using the Shift key, one may highlight more than one Service and use either the right-click menu or the Delete key to delete all selected Services.

    Deleting an item from the SDR Root cannot be undone

21.1.5 Waveforms

Mouse-hovering over an item within the SCA Explorer will show its corresponding file and the full path to its location. Mouse-hovering over a Waveform will show that its representation is derived from the SAD file within the SDR Root.

One may perform the following actions from the SCA Explorer view on SCA Waveform objects:

  • View within SCA Editor: To open the SAD file within an SCA Editor one may either, double-click the SAD file, right-click and select SCA Waveform Editor or right-click and select Open With> SCA Waveform Editor. This will open the SCA Waveform Editor, as described in Section 19.4.1, in read-only mode, one may not edit the fields or add any additional Components.

By selecting these objects with the Properties View open, one can view all the information from the XML in a tabular format.
  • Delete: Delete a Waveform from the SDR Root through the right-click Delete menu. One may also use the keyboard’s Delete key to initiate a deletion. This will remove only the SAD file and the folder containing it from the SDR and will not delete the referenced Components. The delete action also supports multiple selection. Using the Shift key, one may highlight more than one Node and use either the right-click menu or the Delete key to delete all selected Waveforms.

    Deleting an item from the SDR Root cannot be undone

21.2 Browsing Installed IDL Libraries

Even though the IDL Repository is displayed within the Target SDR tree structure, the deployed IDLs and the core IDL library are not stored within the SDR Root directory on the file system. All IDLs exist within the OSSIEHOME directory. However, the IDL repository is shown here so that one may browse the content and view the installed IDLs.

Expanding the IDL Repository tree structure will expose the individual IDLs and a list of module folders which contain IDLs. To view the content of an IDL in a text editor, one may either double-click the IDL or right-click and select Text Editor or Open With…> Text Editor.

One may also expand an IDL. When an IDL is expanded, the IDE will parse its content to decompose it into referenced methods and objects. By selecting these methods and objects with the Properties View open one can view all the information from the IDL in a tabular format.

21.3 Getting Details About Error Conditions

If an error condition occurs within the SDR Root that the IDE can detect, it will mark the object in error with a decorator in the lower left corner. Mouse hovering over the items icon will provide a short description of the issue; however, if more than one problem has occurred the hover text will read “Multiple Problems exist with this item”.

More detail about an error can be found within the Properties View of the item.
To view the details about an error condition:

  1. With the item selected, select or open the Properties View.
  2. From the Properties View, select the Advanced tab
  3. Select the status row. This should cause the Details key to appear.
  4. Press the Details button to bring up a detailed dialog of the current error conditions
