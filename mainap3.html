Appendix C
REDHAWK Provided Plug-ins

The REDHAWK IDE is composed of many plug-ins that serve different purposes. The following tables describe the REDHAWK provided plug-ins.

C.2 gov.redhawk.core Plug-ins

The following table explains the gov.redhawk.core plug-ins, which are a set of common plug-ins for REDHAWK interaction.

Plug-in ID

Description

gov.redhawk.bulkio.orb.omni

Tie in for creating BulkIO CORBA Interface objects that use omniOrb for the CORBA implementation.

gov.redhawk.bulkio.util

Utility classes for creating / using BulkIO CORBA Interface objects.

gov.redhawk.configurator

Open Services Gateway initiative (OSGi) class loader to dynamically load plug-ins that are stored within the SDR root.

gov.redhawk.core.filemanager

An SCA CF CORBA File Manager implementation and service.

gov.redhawk.core.resourceFactory

An SCA CF CORBA Resource Factory implementation and service.

gov.redhawk.interceptor.client

When enabled, intercepts CORBA invocation. It then validates if the invocation was made from a UI / Model locking context. This is to assist in DEBUGGING an application and making sure all CORBA classes don’t block critical threads. Refer to the readme within the plug-in for additional details.

gov.redhawk.sca

Handles the loading and storing of the Domain Manager Registry.

gov.redhawk.sca.efs

An Eclipse File System (EFS) URI provider for SCA file systems. NOTE: Although not directly required, this plug-in should be included in all RCP-based applications.

gov.redhawk.sca.model

The main REDHAWK model. Use this model for all interactions to backend REDHAWK resources.

gov.redhawk.sca.model.edit

UI elements for the main REDHAWK runtime model.

gov.redhawk.sca.model.provider. event

Updates the main REDHAWK model in response to CORBA Event Channel messages.

gov.redhawk.sca.model.provider. refresh

Updates the main REDHAWK model by polling the backend CORBA objects for state.

gov.redhawk.sca.util

Set of utility classes for interacting with the REDHAWK main model.

nxm.redhawk

NeXtMidas Option tree for piping REDHAWK BulkIO data flows into NeXtMidas pipes.
